CHAPTER 01
ภาพรวมระบบ LINE ทั้งหมด
ทำงานยังไง เชื่อมกันยังไง ใครใช้อะไร

ระบบ LINE ของทีม OPS ประกอบด้วย 3 ส่วนหลักที่ทำงานร่วมกัน

┌──────────────────────────────────────────┐ │ ระบบ LINE ทีม OPS │ └──────────────────────────────────────────┘ Google Sheet ← database กลาง ข้อมูลงานทั้งหมด │ ▼ Apps Script ← API กลาง เชื่อม Sheet ↔ LINE │ ┌──┴──────────────┐ │ │ ▼ ▼ LINE LIFF App LINE Bot ถั่วดู Dashboard แจ้งเตือนทีมอัตโนมัติ approve proof ส่ง Flex Message รายบุคคล รับ reply → อัพเดท Sheet
3 ส่วนหลัก
1
Google Sheet — ฐานข้อมูลกลาง
เก็บข้อมูลโปรเจค งาน collections ทุกอย่าง ถั่วอัพเดทที่นี่ ทีมไม่ต้องแตะ
2
LINE Bot — แจ้งเตือนอัตโนมัติ
Bot ส่ง Flex Message หาทีมรายบุคคล ทีม reply ใน LINE → Bot จดแล้วอัพเดท Sheet กลับ ทีมไม่ต้องเปิดแอพอื่น
3
LINE LIFF — Dashboard / Proof Tool
Web app เปิดใน LINE โดยตรง ถั่วใช้ดู overview เพิ่มงาน approve proof ทีมเปิดดูสถานะงานตัวเองได้
CHAPTER 02
LINE LIFF คืออะไร
และถั่วกับทีมเห็นหน้าตาต่างกันได้ยังไง

LIFF = LINE Front-end Framework

คือ web app ที่เปิดอยู่ใน LINE โดยตรง ไม่ต้องออกจาก LINE ไปเปิด browser อื่น กดปุ่มใน Rich Menu แล้วหน้าต่างเล็กขึ้นมาข้างใน LINE เลย

LINE ส่ง userId ของคนกดปุ่มไปให้ LIFF App โดยอัตโนมัติ เช่น userId ของพี่เฟียน vs พี่พีท

✨
แสดงหน้าตาตาม role
ถั่ว → เห็น Admin Dashboard ครบทุกฟีเจอร์ · ทีม → เห็นแค่งานของตัวเอง
ถั่ว vs ทีม — เห็นอะไรต่างกัน
Feature 👩‍💼 ถั่ว (Admin) 👥 ทีมงาน
หน้าแรก Boss Summary + Stats ทั้งบริษัท งานของฉัน (เฉพาะที่ assign ให้ตัวเอง)
Projects เห็นทุก project ทั้ง CC/BG/TR เห็นเฉพาะ project ที่ตัวเองอยู่ด้วย
Proof/Approve กด Approve / Request Revision ได้ ดูสถานะได้ กด approve ไม่ได้
เพิ่มงาน ✅ มีปุ่ม + เพิ่มงานใหม่ ❌ ไม่มี
อัพเดท status อัพเดทได้ทุก task อัพเดทได้แค่ task ตัวเอง ผ่าน Bot reply
💡
วิธีหา LINE userId ของทีม
ให้ทุกคนส่งข้อความหา Bot ก่อนครั้งแรก Bot จะบันทึก userId อัตโนมัติ แล้วถั่วไปกรอก role ใน Sheet

CHAPTER 05
คู่มือทีมงาน
ทีมทำอะไรใน LINE บ้าง — ทำได้ง่ายมาก
👥 ทีมงาน (Read-only + Bot Reply)
✅
ทีมไม่ต้องเรียนรู้เครื่องมือใหม่
ทุกอย่างทำใน LINE ที่ใช้อยู่แล้ว · Bot ถาม → reply ใน chat เดิม · ไม่ต้องเปิดแอพอื่น
วิธี reply Bot ที่ถูกต้อง
คำที่ Bot เข้าใจ: ✅ "เสร็จ" หรือ "เสร็จแล้ว" หรือ "done" → Bot อัพเดท status = done 🟡 "ยัง" หรือ "ยังไม่เสร็จ" หรือ "inprogress" → Bot บันทึกว่ายังทำอยู่ 🔴 "ติดปัญหา" หรือ "blocked" หรือ "ติด" → Bot แจ้งถั่วทันที ✅ "รับทราบ" หรือ "ok" หรือ "โอเค" → Bot บันทึกว่ารับทราบแล้ว 💡 กดปุ่มใน Flex Message → Bot รับอัตโนมัติ ไม่ต้องพิมพ์
⚠️
ถ้าติดปัญหาจริงๆ
พิมพ์ "ติดปัญหา" แล้วอธิบายสั้นๆ ว่าติดอะไร Bot จะส่งให้ถั่วพร้อม context ทันที ไม่ต้องกลัวว่าจะกวนใจ — ถั่วอยากรู้เร็วกว่าค้างงาน
